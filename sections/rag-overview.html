Block 4 — End to End Platform Components enabling KYB

RAG Architecture Overview

Why Retrieval-Augmented Generation is the only appropriate pattern for compliance-critical KYB — and how it's designed here.

Core Principle
A pure LLM is not sufficient for KYB. The task is evidence-based, policy-bound, and audit-sensitive. The model is not the source of truth — retrieved policy text, applicant documents, and reviewer-approved rules are.
Traceable Auditable Policy-aligned Reviewer-verifiable Compliance-safe

Why Not a Pure LLM

Pure LLM — 3 failure modes
  • Hallucination — fabricates plausible-sounding policy or facts with no grounding
  • Staleness — training cutoff means policy changes go undetected
  • Non-traceability — no evidence link means audit is impossible
RAG — what it fixes
  • Retrieves current policy + case evidence before generating
  • Grounds every finding in retrievable source evidence
  • Attaches explicit citations to every claim and recommendation
  • Abstains when evidence is weak, missing, or conflicting
Fact-checking
Evidence vs submission
Policy application
Rules, not model memory
Gap identification
Missing or inconsistent docs
Audit chain
Evidence lineage preserved
Generate only after retrieving.  Recommend only with evidence.  Escalate when evidence is weak, conflicting, or missing.

RAG System Components

Retrieval Layer
Retriever
Semantic search over policy corpus and case document corpus
Vector DB + Embedding Model
Metadata Filter Layer
Narrows retrieval by jurisdiction, entity type, policy version, case stage
Structured Metadata Filtering
Ranker / Re-ranker
Re-scores chunks to improve precision and reduce noisy context
Cross-encoder + BM25 Hybrid
Query Builder
Converts workflow step / criterion into targeted retrieval queries
Prompted Query Expansion
Context Assembler
Builds grounded context package — chunk selection, deduplication, token-budget management
Custom Chunk Assembly
Generation Layer
Generator
Produces structured findings and recommendations using retrieved evidence only
LLM + Schema-constrained Output
Citation Engine
Links every claim, finding, and recommendation to source chunks with chunk IDs
Custom Post-processing
Safety & Trust Layer
Confidence Engine
Calibrated signal from evidence strength, retrieval quality, coverage, and model certainty
Heuristic + Model-assisted Scoring
Abstention / Fallback Layer
Prevents unsupported output when evidence is weak, conflicting, or absent
Rule Thresholds + Fallback Policies
Operations Layer
Audit Logger
Immutable log — retrieval set, prompt version, output, citations, reviewer edits, final decision
Immutable Decision Log / Audit Store
Knowledge Management
Ingestion, chunking, versioning, approval, and retirement of policy content
Policy CMS + Document Pipeline
Evaluation Layer
Measures retrieval quality, citation quality, grounding, and reviewer acceptance
Offline Eval Suite + Prod Telemetry

What RAG Should Not Do

Autonomously approve or reject cases without human controls
Answer from general world knowledge
Infer facts absent from submitted case documents
Invent rationale for missing evidence
Replace policy ownership by Compliance
Operate without versioned knowledge sources
Produce uncited persuasive prose
Design Principle
The KYB Copilot is an evidence-grounded compliance assistant — not a freeform chatbot.
Retrieve · Organise · Synthesise → structured recommendation a reviewer can validate, edit, or reject.
