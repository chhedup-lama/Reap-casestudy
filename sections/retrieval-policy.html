1What This Layer Does

Responsibilities
  • Read scanned and digital documents
  • Extract text and preserve layout
  • Identify document type
  • Detect important structural elements
  • Extract key fields and entities
  • Prepare retrieval-friendly chunks
  • Run quality checks before downstream use
Not in Scope Here
  • Apply policy rules
  • Generate recommendations
  • Perform case-level reasoning
  • Make workflow decisions
These come in the next layer — Knowledge, Retrieval & Policy Grounding.

3Core Modules

OCR & Parsing
Does
  • Extracts text from native and scanned documents
  • Preserves page and layout structure
  • Detects tables and key-value regions
  • Orientation correction and image pre-processing
  • Confidence scoring at page / block level
Output preserves layout-aware structure — not flat text only — so findings trace back to source location.
Foundation for field extraction, chunking, citations, evidence inspection, and reviewer trust.
Doc Classification
Does
  • Identifies document type
  • Routes file to the appropriate extraction schema
  • Distinguishes document-specific handling logic
A certificate of incorporation, bank statement, and director ID should not use the same extraction logic. Classification is the control point that makes the pipeline intelligent rather than generic.
Field Extraction
Does
  • Extracts key fields from classified documents
  • Normalises business and compliance attributes
  • Converts layout-aware content into structured signals
Company nameRegistration number Incorporation dateAddress Director namesUBO names Ownership %Issue / expiry dates
Converts documents from readable files into usable case evidence.
Validation & Completeness
Does
  • Checks whether the document is usable
  • Validates expected fields are present
  • Flags low-confidence or incomplete outputs
  • Gates documents before downstream retrieval and review
Prevents weak extraction from silently degrading retrieval, reasoning, and reviewer outputs.

4OCR & Text Extraction Requirements

Page-level extraction
Layout preservation
Table and key-value block detection
Multi-language support where relevant
Orientation correction
Image pre-processing for poor scans
Signature / stamp region preservation
Confidence scoring at page / block level
Output must preserve structural fidelity so downstream modules can extract fields accurately, map findings to source, support citations, and improve reviewer inspection.

5Structure Extraction

After OCR, the system detects document-aware structure — not a flat page of text. This is the bridge between raw OCR output and meaningful document understanding.

Structures Detected
Section headersOwnership tables Address blocksDates & identifiers Authority & registration details Signatures / sealsDocument-specific field regions
Improves
  • Extraction accuracy
  • Schema validation

  • Retrieval precision
  • Reviewer usability

6Chunking Strategy

Chunking is the handoff point from Document Intelligence to Knowledge, Retrieval, and Policy Grounding. Poor chunking cascades into every downstream layer.

Chunking Design Requirements
  • Preserve semantic boundaries
  • Avoid splitting critical clauses or evidence mid-thought
  • Keep chunks citation-friendly
  • Retain document and page provenance
  • Handle tables and lists carefully
  • Separate unrelated sections
  • Allow limited overlap for context continuity
Chunking Inputs
Page structureSection boundaries Document typeTable regions Paragraph flowToken budget constraints
Poor Chunking Leads To
  • Weak retrieval
  • Low-quality grounding
  • Broken citations
  • Fragmented evidence
  • Lower reviewer trust

7Quality Checks

System Quality Checks
  • File integrity and format validation
  • OCR confidence threshold
  • Text extraction non-emptiness
  • Page count consistency
  • Duplicate file detection
  • Doc type classification confidence
  • Schema-level completeness check
  • Field extraction sanity checks
  • Chunk generation success
  • Metadata completeness check
Reviewer-Facing Quality Warnings
  • Illegible scan
  • Truncated upload
  • Unsupported format
  • Likely wrong document type
  • Missing expected fields
  • Conflicting identifiers across documents
These make quality visible early — before weak documents cause errors in retrieval, citation, reasoning, or review.
