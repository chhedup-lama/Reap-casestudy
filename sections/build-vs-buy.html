Block 3 — Product Scope

Build vs Buy Analysis

Make/buy decision across every layer of the KYB AI stack — rationale anchored in differentiation, not cost alone.

Decision Framework

💡
Principle: Buy commodity infrastructure. Build differentiated workflow, policy enforcement, and HITL layers. Never outsource competitive moat.

Data Ingestion Layer

Document Parsing & Chunking
Build Core
Chunking strategy, evidence mapping, citation structure
Parsing / document AI vendors to accelerate
Chunking and evidence mapping must reflect KYB retrieval and citation needs — not generic
Document Classification
Build Core
Business-specific doc taxonomy, routing logic
Document AI tools for base classification
Base classification can be accelerated externally, but KYB taxonomy and routing must be yours
Rules & Decisioning Engine
Build Core
Internal rules framework (lightweight, fast to build)
BPM / rules engine if non-engineers must manage large rule sets
Lightweight rules are fast to build; configurable tooling only if scale demands it
Governance & Access Control
Build Core
App-level authorization model, reviewer permissions, tenant scoping
IAM / RBAC providers for identity foundations
Identity infra can be bought, but application-specific access and reviewer permissions must be internal
Case Orchestration / Workflow Engine
Build Core
Case-state model, exception logic, stage transitions
BPM / workflow tools for execution layer
Workflow engine can be bought, but case logic and state transitions are your product
